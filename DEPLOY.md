# 私有化部署说明（DEPLOY.md）

本系统支持完全私有化部署：整套服务跑在你公司自己的服务器上，上传的文档和对话记录全程不出内网。下面是三种数据安全档位，按你的合规要求选一种。

## 前置条件

一台能装 Docker 的服务器（公司内网的 Linux 机器即可），安装 Docker 和 Docker Compose。不需要联网下载文档数据，只在构建镜像时需要一次网络下载依赖。

## 一行命令启动

把项目代码放到服务器上，进入项目目录，执行：

```bash
docker compose up -d
```

启动后访问 `http://服务器IP:8000` 即可使用。停止用 `docker compose down`。

## 三种数据安全档位

### 档位一：完全离线（最高安全）

什么 API Key 都不配置。系统以纯检索模式运行：

- 不向任何外部服务发送请求，连公网都不需要连
- 文档只存在服务内存里，服务重启即清空，磁盘上不留痕
- 回答方式是返回文档中最相关的原文片段（不做 AI 总结）

适合：对数据安全要求极高、不允许任何数据离开内网、连云端大模型调用都禁止的客户（如涉密、金融、政企）。

### 档位二：接云端大模型（DeepSeek）

在 `docker-compose.yml` 里填入 `DEEPSEEK_API_KEY`。提问时，系统会把检索到的文档片段发送给 DeepSeek 做总结，生成自然语言回答。

适合：可以接受调用国内合规云端大模型、希望回答更自然流畅的客户。注意：文档片段会经过 DeepSeek 服务器，需客户知情同意。

### 档位三：完全本地化的智能问答（数据零外发 + AI 总结）

这是兼顾"智能回答"和"数据不出内网"的方案。客户在内网自建一个开源大模型服务（例如用 vLLM 部署 Qwen），然后在 `docker-compose.yml` 里配置：

```
OPENAI_API_KEY=任意占位字符串
OPENAI_BASE_URL=http://内网模型地址:8000/v1
OPENAI_MODEL=你部署的模型名
```

这样提问时，文档片段只发往公司内网的模型服务，AI 总结全过程在内网完成，一个数据包都不出公司网络，同时还能享受智能问答的体验。

适合：既要 AI 智能总结、又绝对不允许数据外发的客户。这是纯云端产品（如各家官网的文档问答）做不到的。

## 验证部署成功

启动后访问 `http://服务器IP:8000/api/health`，返回 `{"status":"ok","live_mode":...}` 即表示服务正常。`live_mode` 为 true 表示智能问答模式已启用，false 表示纯检索模式。

## 数据与隐私

- 上传文档内容只存在服务进程内存，按会话隔离，不写入磁盘或数据库
- 服务重启或会话超时（默认 2 小时）后自动清空
- 如需持久化存储，需另行加密落盘并做客户级隔离（当前版本未实现，属下一步工程）
